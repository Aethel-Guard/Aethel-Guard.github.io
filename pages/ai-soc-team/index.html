// The Team

Four AI agents. One mission.

Each agent has a specialized role. They communicate over NATS message bus and coordinate through a shared event pipeline.

α

Alpha

Detection & Triage

Alpha monitors incoming events from Wazuh SIEM and other sensors. It correlates alerts, deduplicates noise, and ranks threats by severity using the AI inference engine. When a genuine threat is confirmed, Alpha escalates to Bravo for response.

  • Alert correlation
  • Severity scoring
  • False-positive filtering
  • Threat classification
  • AI triage
β

Bravo

Response & Remediation

Bravo receives confirmed incidents from Alpha and executes response playbooks through Shuffle SOAR. It can isolate hosts, block IPs, revoke credentials, and trigger automated containment. Bravo logs every action for audit and can escalate to human analysts when needed.

  • SOAR playbooks
  • Host isolation
  • IP blocking
  • Credential revocation
  • Human escalation
γ

Charlie

Threat Intelligence

Charlie pulls and enriches threat intelligence from OpenCTI and MISP. It maintains IOC feeds, tracks adversary TTPs, and provides context to Alpha's detections and Delta's investigations. Charlie syncs indicators bidirectionally with external sharing communities.

  • OpenCTI enrichment
  • MISP sharing
  • IOC management
  • TTP mapping (MITRE ATT&CK)
  • Feed synchronization
δ

Delta

Digital Forensics

Delta performs deep-dive forensic analysis on confirmed incidents. It collects disk images, memory dumps, and network captures, then correlates artifacts using timeline analysis. Delta writes findings directly into TheHive case files and produces court-admissible reports.

  • Memory analysis
  • Disk forensics
  • Network capture
  • Timeline reconstruction
  • TheHive case management

// Quick Start

Deploy in minutes

Clone, configure, and launch your SOC with Docker Compose. Full instructions in the Integration Guide.

bash 
# Clone the repository
git clone https://github.com/Aethel-Guard
cd Aethel-Guard
bash 
# Configure your environment
cp .env.example .env
# Edit .env — set passwords, API keys, tenant config
bash 
# Launch Core tier
docker compose -f docker-compose.core.yml up -d

# Or launch Full tier
docker compose -f docker-compose.full.yml up -d
bash 
# Verify the stack is healthy
curl -s http://localhost:8080/health | jq .

# Open the dashboard
# → http://localhost:4200 (Angular UI)
