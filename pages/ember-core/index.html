// Security Architecture

Selene — the guard that cannot be bypassed

Selene is architecturally isolated from the Companion model. Compromising the Companion does not compromise Selene. Guards are hardcoded at inference time — no prompt override is possible.

User Input

Prompt / query

↓

Selene Input Guard

Intercepts jailbreak attempts
BEFORE Companion processes

↓ Passed

Companion (Ember Prime)

Generates response
Subject to P1-P7 constraints

↓

Selene Output Guard

Blocks security-relevant outputs
AFTER generation

↓ Approved

Response

Delivered to user

P1 — P7 Hardcoded

Seven security principles enforced at inference time. No admin, developer, or debug backdoor commands exist. No prompt can override them.

Architectural Isolation

Selene runs in a separate process. Compromising the Companion model does not compromise Selene. The guard has its own execution context.

Adversarial Training

10× oversample against all 10 jailbreak categories. Selene detects drift toward compliance with jailbreaks and aborts immediately.

Self-Monitoring

Selene continuously monitors its own behavior. If it detects drift toward fulfilling a jailbreak request, it aborts the operation automatically.

S1-S10 at Inference

Ten safety constraints hardcoded at inference time. Not part of the prompt — baked into the inference pipeline itself. No prompt override possible.

Applies to All Models

Selene guards apply across the entire model family: Selene itself, Companion (Ember Prime), and Ember Edge. No model runs unguarded.

// Inference Server

FastAPI inference on port 8081

Every Aethel Guard product routes inference requests through the Ember Core FastAPI server. Product-specific system prompts ensure correct behavior per product.

Server Architecture

  • ✓ FastAPI async inference server
  • ✓ Port 8081 — dedicated to Ember Core
  • ✓ Selene guards run on every request
  • ✓ Product-specific system prompts (Vigilar vs Guard)
  • ✓ No Ollama dependency — direct ember-server
  • ✓ GGUF deployments: Q4_K_M, Q5_K_M, Q8_0

python 
# Ember Core inference server — product routing
from fastapi import FastAPI
from selene import InputGuard, OutputGuard

app = FastAPI(title="Ember Core")

PRODUCT_PROMPTS = {
    "vigilar": "You are a consumer security companion...",
    "vigilar-guard": "You are a detection and triage engine...",
    "ai-soc": "You are a SOC analyst...",
}

@app.post("/v1/chat")
async def chat(product: str, message: str):
    # Selene input guard — intercepts jailbreaks
    InputGuard.check(message)

    prompt = PRODUCT_PROMPTS.get(product, "")
    response = ember_prime.generate(prompt + message)

    # Selene output guard — blocks sensitive outputs
    OutputGuard.check(response)

    return {"response": response}
