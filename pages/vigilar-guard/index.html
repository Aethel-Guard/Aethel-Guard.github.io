// SIEM Integrations

Forward to everything

Guard doesn't lock you in. Alerts flow to your existing SIEM stack through standard protocols.

🟠

SIEM

Webhook (Bidirectional)

Full bidirectional integration. Guard sends alerts to your SIEM and receives events for correlation.

  • Alert webhook to SIEM
  • Event ingestion from active responses
  • Correlation with rule engine
  • Shared MITRE ATT&CK mapping
🔵

Threat Intel

STIX 2.1 Push

Guard pushes enriched threat intelligence as STIX 2.1 bundles. Observable-driven, relationship-aware.

  • STIX 2.1 bundle export
  • Indicator, observable, and relationship objects
  • Confidence score propagation
  • Pull enrichment from connected threat intel platforms
🔴

Threat Sharing

STIX/TAXII + Community Feeds

Guard shares detections with threat communities automatically. Attributes, tags, and taxonomies.

  • STIX/TAXII event creation
  • Attribute tagging with Galaxy clusters
  • Automatic IOC sharing to communities
  • Feed synchronization support
📄

Splunk / QRadar / ArcSight

CEF / Syslog Forward

Common Event Format and syslog forwarding to any SIEM that supports industry-standard ingestion.

  • CEF-formatted alert messages
  • Standard syslog (RFC 5424) transport
  • TCP + UDP delivery
  • Splunk, QRadar, ArcSight, rsyslog compatible
// API Endpoints

FastAPI on port 8082

RESTful API for event ingestion, alert retrieval, and health monitoring. OpenAPI 3.1 spec included.

POST /api/v1/events Ingest a security event for detection and triage
GET /api/v1/alerts List recent alerts with severity and status filters
GET /api/v1/alerts/{alert_id} Get full alert detail with enrichment and triage explanation
POST /api/v1/rules Upload or update detection rules
GET /api/v1/rules List all active detection rules and their categories
GET /api/v1/triage/status Check AI triage availability and current model
POST /api/v1/integrations/wazuh Configure SIEM webhook endpoint
POST /api/v1/integrations/opencti Configure threat intel STIX 2.1 push target
POST /api/v1/integrations/misp Configure threat sharing push target
GET /api/v1/health Service health check with dependency status
WS /ws/v1/alerts/stream Real-time alert stream via WebSocket
