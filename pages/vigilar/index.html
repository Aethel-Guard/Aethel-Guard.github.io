// The Problem

Consumer security is broken

The apps you trust to protect you are harvesting your data. "Free" security products trade your privacy for profit. The industry calls it security. We call it what it is.

📡

Tracking as a Business Model

"Free" security apps embed analytics SDKs, advertising identifiers, and fingerprinting scripts. They block viruses while tracking you themselves. The fox guarding the henhouse — and the henhouse has cameras.

☁️

Cloud Dependency

Every security decision routes through someone else's server. Your files, your messages, your browsing — all processed in the cloud. When that server breaches, your most sensitive data goes with it.

🔓

Policies, Not Guarantees

Privacy policies are promises, not code. They change at the next update. They have exceptions for "lawful requests." They have clauses you'll never read. What you need is architecture, not paperwork.

// P1–P7 Redlines

Seven lines that cannot be crossed.

These are not policies. They are not preferences. They are hardcoded constants in the source — immutable, non-negotiable, and enforced at compile time. No prompt, no privilege, no update can override them.

P1

No PII Exfiltration

Personally identifiable information is never transmitted off-device. No telemetry, no analytics, no crash reports containing user data. The network layer physically cannot access PII buffers.

P2

No Jailbreak Bypass

Redlines cannot be overridden by any instruction — user, admin, or injected. There is no debug mode, no developer backdoor, no emergency override. The code path to bypass does not exist.

P3

No Model Substitution

The on-device AI model cannot be swapped, replaced, or augmented by an external model. Ember Edge is compiled in. No remote model loading, no model marketplace, no silent replacement.

P4

No Silent Failover

If a security component fails, it fails closed — not open. Every failover creates an auditable record. No silent degradation, no fallback to insecure mode, no quiet bypass.

P5

No Unapproved External Calls

Vigilar does not make network calls unless explicitly approved by the user through MFA. No background sync, no silent update checks, no beacon pings. Every outbound connection is intentional and logged.

P6

No Data Retention Beyond Session

Sensitive session data is wiped from memory when the session ends. No caches, no temp files, no persistent logs. What was processed is gone — not recoverable, not reconstructable, not subpoena-able.

P7

No Reduced-Thinking Execution

The AI inference engine cannot be forced into a reduced-thinking or low-effort mode. Security analysis always runs at full capability. No "quick mode," no "lite mode," no performance shortcut that sacrifices security.

// On-Device AI

Ember Edge. AI on your device.

Vigilar's intelligence comes from Ember Edge — a 3.1B parameter model running entirely on your device. No cloud inference. No API calls. No data leaving your phone. Just local intelligence, always available, always private.

🔥

Zero cloud. Zero API calls. Zero data sent.

Ember Edge is a quantized 3.1B parameter model (Q4_K_M) that runs directly on your device's CPU/GPU. It handles threat analysis, behavioral anomaly detection, phishing URL classification, and adversarial input detection — all without a single network round-trip. The model is embedded in the app binary. It cannot be replaced, augmented, or bypassed.

  • ✓ 3.1B parameters, Q4_K_M quantization — runs on mobile hardware
  • ✓ Threat classification, anomaly detection, phishing analysis
  • ✓ Adversarial input detection and prompt injection defense
  • ✓ P3 redline: model cannot be substituted or remotely loaded
  • ✓ P7 redline: full capability always — no reduced-thinking mode
  • ✓ Inference runs in-process — no external server, no API endpoint
